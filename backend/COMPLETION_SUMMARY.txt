================================================================================
                              KEY FEATURES
================================================================================

AUTHENTICATION
- JWT-based Bearer token authentication
- Bcrypt password hashing with salt
- 30-minute token expiration (configurable)
- Protected endpoints with Depends(get_current_user)

DATABASE
- Async SQLAlchemy 2.0 with PostgreSQL
- UUID primary keys on all tables
- Proper indexing on search columns (part_num, set_num, email)
- Unique constraints to prevent duplicates
- Cascade deletes for data integrity
- Connection pooling (20 connections, 10 overflow)

CACHING
- Redis integration with async client
- Parts search: 1 hour TTL
- Set data: 24 hours TTL (rarely changes)
- Cache invalidation patterns implemented

LEGO PIECE RECOGNITION
- Primary: Local ONNX model inference (optional)
  - Image preprocessing with Pillow (224x224 resize, normalization)
  - Returns top 3 predictions with confidence scores
- Fallback: Google Gemini 2.0 Flash Vision API
  - Base64 image encoding
  - JSON parsing of predictions
  - Cross-reference with database

INVENTORY MANAGEMENT
- Add/update/delete pieces from personal collection
- Track part number, color, quantity
- CSV export of entire inventory
- Unique constraint on (user_id, part_id, color_id)

SET BUILDING
- Compare inventory against official LEGO sets
- Calculate missing pieces
- Show completion percentage
- Generate BrickLink wanted list in XML format

BRICKLINK INTEGRATION
- Color mapping (Rebrickable ↔ BrickLink)
- XML wanted list generation (ImportLIable format)
- Missing parts calculation

EXTERNAL INTEGRATIONS
- Rebrickable API: Part/set catalog, search
- Google Gemini API: Vision-based identification
- AWS S3: Image storage (configured)
- BrickLink: Wanted list export

                           API ENDPOINTS SUMMARY
================================================================================

PUBLIC ENDPOINTS
  POST   /auth/register             - Create account
  POST   /auth/login                - Get JWT token
  GET    /parts?search=              - Search parts (cached)
  GET    /parts/{part_num}           - Part details (cached)
  GET    /sets?search=               - Search sets (cached)
  GET    /sets/{set_num}             - Set details (cached)
  GET    /sets/{set_num}/parts       - Parts in set
  GET    /bricklink/colors           - Color mapping
  GET    /health                     - Health check

PROTECTED ENDPOINTS (require JWT)
  GET    /auth/me                    - Current user info
  GET    /inventory                  - User's collection
  POST   /inventory                  - Add piece (upserts on duplicate)
  PUT    /inventory/{id}             - Update quantity
  DELETE /inventory/{id}             - Remove piece
  GET    /inventory/export           - CSV export
  POST   /scan                       - Identify piece from image
  POST   /scan/confirm               - Confirm identification
  POST   /bricklink/wanted-list/{set_num}  - Generate BrickLink XML
