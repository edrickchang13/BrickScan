API INTEGRATION
================================================================================

All endpoints fully integrated:

Authentication:
  POST /auth/login
  POST /auth/register

Scanning:
  POST /scan (base64 image → predictions)

Inventory:
  GET /inventory
  POST /inventory
  PATCH /inventory/{id}
  DELETE /inventory/{id}
  GET /inventory/export/csv

Sets:
  GET /sets/search
  GET /sets/{setNum}
  GET /sets/{setNum}/parts

Build Check:
  POST /builds/check

BrickLink:
  POST /bricklink/wanted-list

See API_ENDPOINTS.md for full specifications.

================================================================================
STATE MANAGEMENT ARCHITECTURE
================================================================================

Auth State (Zustand):
  - user: User | null
  - token: string | null
  - isLoading: boolean
  - isLoggedIn: boolean
  - Actions: login(), register(), logout(), loadStoredAuth()
  - Persistence: expo-secure-store

Inventory State (Zustand):
  - items: InventoryItem[]
  - isLoading: boolean
  - error: string | null
  - Actions: fetchInventory(), addItem(), updateItem(), removeItem()
  - Optimization: Optimistic updates with rollback on error

Server State (TanStack Query):
  - Sets search results
  - Set details
  - Build check results
  - Automatic caching & refetching

================================================================================
KEY DESIGN DECISIONS
================================================================================

1. Tab Navigation (4 tabs):
   - Each tab has independent navigation stack
   - Better discoverability than drawer navigation
   - Standard iOS pattern

2. Zustand for Auth & Inventory:
   - Simple, lightweight state management
   - Secure token persistence
   - Optimistic updates for better UX

3. TanStack Query for Server State:
   - Automatic caching & refetching
   - Built-in error handling
   - Efficient network usage

4. NativeWind Styling:
   - Consistency across all screens
   - Type-safe Tailwind classes
   - Rapid development

5. Image Base64 for Scanning:
   - Simpler than file uploads
   - Works well with existing APIs
   - No multipart complexity

6. Modal Quantity Selector:
   - Larger touch targets
   - Clear quantity feedback
   - Standard iOS pattern

7. Optimistic Updates:
   - Instant feedback to user
   - Rollback on error
   - Perceived performance
