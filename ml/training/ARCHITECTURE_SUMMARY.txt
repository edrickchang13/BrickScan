ARCHITECTURE OVERVIEW
================================================================================

DUAL-HEAD EFFICIENTNET-B3 CLASSIFIER
   Input: 224x224 RGB images (ImageNet normalized)
   |
   ├─ Pretrained EfficientNet-B3 Backbone (features: 1536 dim)
   |
   ├─ Part Head:  FC(1536→512) + ReLU + Drop(0.4) → num_parts logits
   |
   └─ Color Head: FC(1536→256) + ReLU + Drop(0.3) → num_colors logits
   |
   └─ ONNX Export: Concatenate → [batch, num_parts + num_colors]

TRAINING PIPELINE COMPONENTS:
   1. Data Loading & Augmentation (dataset.py)
      - Random resized crop, horizontal flip, color jitter, rotation
      - Separate training and validation augmentations
      - ImageNet normalization

   2. Model Definition (model.py)
      - Dual-head architecture with shared backbone
      - forward() for training
      - forward_onnx() for export

   3. Training Loop (train.py)
      - Mixed precision (torch.cuda.amp)
      - Gradient clipping (norm=1.0)
      - AdamW optimizer with weight decay (1e-5)
      - OneCycleLR scheduler
      - Early stopping (patience=10)
      - TensorBoard logging
      - Checkpointing every 5 epochs

   4. ONNX Export (export_onnx.py)
      - torch.onnx.export with opset_version=14
      - ONNX model validation
      - onnxruntime inference test

   5. Evaluation (evaluate.py)
      - Part top-1 and top-5 accuracy
      - Color top-1 accuracy
      - Confusion matrix for top-20 parts
      - Results saved to JSON

FASTAPI INTEGRATION
================================================================================

The ONNX model is specifically designed for FastAPI inference:

INPUT FORMAT:
  - Tensor shape: [batch_size, 3, 224, 224]
  - Data type: float32
  - Normalization: ImageNet (mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225])

OUTPUT FORMAT:
  - Tensor shape: [batch_size, num_parts + num_colors]
  - Data type: float32 (raw logits, no softmax)
  - Splitting logic:
    * part_logits = output[:, :num_parts]
    * color_logits = output[:, num_parts:]

INFERENCE WORKFLOW:
  1. Load ONNX model with onnxruntime
  2. Load part_labels.json and color_labels.json
  3. Preprocess image → 224x224 → normalize
  4. Run inference
  5. Apply softmax to both heads
  6. Get top-3 parts and best color
  7. Map indices back to labels using JSON encoders

PERFORMANCE EXPECTATIONS
================================================================================

TRAINING TIME (GPU - NVIDIA):
  - 50 epochs with batch_size=64: ~2-4 hours
  - Checkpoint interval: every 5 epochs (~24-30 minutes)

MEMORY REQUIREMENTS:
  - GPU: ~8-10 GB VRAM (EfficientNet-B3 with dual heads)
  - Batch size 64: recommended
  - Mixed precision enabled by default

INFERENCE TIME (onnxruntime):
  - Single image (batch=1): ~20-50 ms on GPU
  - Batch 64: ~100-200 ms total

ACCURACY EXPECTATIONS (typical):
  - Part top-1: 85-92%
  - Part top-5: 94-98%
  - Color top-1: 90-96%
  - Depends on dataset size, class balance, label quality

MODEL SIZE:
  - PyTorch checkpoint: ~150-200 MB
  - ONNX model: ~100-150 MB (smaller after export)
  - Suitable for mobile/edge inference
