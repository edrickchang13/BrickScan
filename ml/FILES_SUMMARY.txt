================================================================================
TRAINING WORKFLOW
================================================================================

1. DATA PREPARATION (one of):
   - Download: python data_pipeline/rebrickable_images.py
   - Render: blender --python data_pipeline/ldraw_renderer.py
   - Augment: python data_pipeline/augmentation.py

2. TRAINING:
   - Configure: Edit training/config.yaml
   - Train: cd training && python train.py
   - Outputs: checkpoints/best_model.pt (+ optimizer/scheduler/class_mapping)

3. EVALUATION:
   - Test: python training/evaluate.py
   - Outputs: eval_results/ (metrics.json + visualizations)

4. EXPORT:
   - To ONNX: python export/to_onnx.py
   - To Core ML: python export/to_coreml.py
   - Outputs: models/ (brickscan.onnx + BrickScan.mlpackage)

5. INFERENCE:
   - Backend: Use inference/predictor.py with ONNX
   - iOS: Use exported Core ML model with Vision framework

================================================================================
KEY METRICS & OUTPUTS
================================================================================

Training Checkpoint (best_model.pt):
  - model_state_dict: EfficientNet-B4 weights
  - optimizer_state_dict: AdamW optimizer state
  - scheduler_state_dict: Cosine annealing state
  - config: Full training configuration
  - class_to_idx: Part number to index mapping (3000 entries)
  - metrics: Best validation metrics
  - epoch: Training epoch when saved

Evaluation Report (evaluation_metrics.json):
  - overall_accuracy: Top-1 accuracy (0-1)
  - top5_accuracy: Top-5 accuracy (0-1)
  - per_class_accuracy: {part_num: accuracy} for all 3000 parts
  - easiest_classes: Top 10 easiest to classify
  - hardest_classes: Top 10 hardest to classify
  - top_confused_pairs: Most confused part pairs with counts
  - calibration: Confidence calibration metrics by bin

Model Exports:
  - brickscan.onnx: ~85 MB ONNX model
  - BrickScan.mlpackage: ~45 MB Core ML model
  - class_mapping.json: {part_num: idx} for all classes
  - model_info.json: Metadata (shape, names, version)
