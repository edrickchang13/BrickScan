YOLOv8 LEGO PIECE DETECTION - START HERE
================================================================================

This directory contains a complete training pipeline for detecting LEGO pieces
using YOLOv8m on NVIDIA DGX Spark GB10.

QUICK START
===========

1. Run the complete pipeline:
   cd ~/brickscan/ml
   python train_yolo_detector.py \
     --parts-dir ~/brickscan/ml/data/test_renders \
     --output-dir ./yolo_detector

2. Or use the shell wrapper:
   ./train_yolo.sh

3. Monitor progress:
   tail -f yolo_detector/training/yolo_detector/results.csv

Expected time: 4-5 hours total

KEY FEATURES
============

Synthetic Data:
  - Generate 5000 training + 1000 validation images
  - No manual annotation needed
  - 4 background types (solid, gradient, noise, texture)
  - Realistic occlusion and overlap

Training:
  - YOLOv8m (medium) architecture
  - Strong augmentation (mosaic, mixup, HSV)
  - Early stopping (patience=15, epochs=100)
  - GPU accelerated

Deployment:
  - ONNX export (framework-agnostic)
  - Training metrics saved
  - Ready for production

COMMAND REFERENCE
=================

Full pipeline:
  python train_yolo_detector.py --output-dir ./yolo_detector

Data only:
  python train_yolo_detector.py --output-dir ./yolo_detector --no-train --no-export

Training only (existing data):
  python train_yolo_detector.py --output-dir ./yolo_detector --no-generate-data

Custom parameters:
  python train_yolo_detector.py \
    --output-dir ./yolo_detector \
    --epochs 200 \
    --batch-size 64 \
    --patience 20

Help:
  python train_yolo_detector.py --help

SUPPORT
=======

Issue                  Solution
==============================================================
Out of memory         → Reduce batch size, image size
Slow training         → Increase batch size (DGX can handle it)
Poor results          → Increase data, extend training
Training interrupted  → Automatically resumes from checkpoint
Missing dependencies  → Script auto-installs ultralytics

All questions answered in documentation files above.
